# SoulCore QClaw Skill 实施计划（MVP）

## 阶段 0：协议冻结（0.5 天）

- 确认接口字段（见 `skill-api-v1.json`）
- 确认会话标识策略：`session_id + user_id`
- 确认最小诊断字段集合

验收标准：

- 协议字段不再频繁变更
- 至少 3 条样例请求/响应可跑通

## 阶段 1：核心服务（1~2 天）

目标：本地服务可独立运行。

工作项：

- 实现 `/v1/chat`
- 实现 `/v1/memory/write`
- 实现 `/v1/memory/recall`
- 实现 `/v1/diagnostics/last`

建议技术栈：

- Python + FastAPI + SQLite（先快跑）

验收标准：

- Postman/curl 可调用 4 个接口
- 能保存并召回记忆
- 能输出基础诊断

## 阶段 2：人格一致性守卫（1 天）

目标：减少角色漂移。

工作项：

- 引入 persona profile（语气/边界/价值观）
- 在回复后做一致性评分（0~1）
- 低分时触发轻量重写（一次）

验收标准：

- 人格偏移场景下，重写后 alignment 分数提升

## 阶段 3：QClaw Skill 适配（1 天）

目标：通过 QClaw 指令调用服务。

工作项：

- Skill 命令模板（例如“记录偏好”“按历史风格回复”）
- 会话上下文映射
- 错误提示友好化

验收标准：

- 微信/桌面端下达指令后可得到稳定回复

## 阶段 4：质量与发布（0.5~1 天）

工作项：

- 增加日志与故障回退
- 输出部署文档（启动、配置、排障）
- 输出演示脚本（3~5 个场景）

验收标准：

- 冷启动可跑
- 连续 30 分钟无崩溃
- 关键日志完整可追踪
