# QClaw Skill 使用说明（极简版）

适合第一次使用的用户：双击一次即可安装并启动本地记忆服务。

仓库地址：

- [https://github.com/youzi0812/qclaw-skill](https://github.com/youzi0812/qclaw-skill)

## 30 秒上手（Windows）

1. 下载仓库到本地
2. 双击根目录 `quick_start.bat`
3. 打开浏览器访问 `http://127.0.0.1:8000/v1/health`

只要返回 JSON 且 `ok=true`，就可以开始使用长期记忆能力。

> 如果 `8000` 被占用，脚本会自动切换到 `8001~8010`，以终端实际显示 URL 为准。

## 手动方式（可选）

### 1) 安装技能

```bash
npx skills add "C:\Users\Administrator\Desktop\游戏插件" -g -y
```

### 2) 启动本地后端

```powershell
cd qclaw-skill
.\start_local.ps1 -Mode noauth
```

## 可选：鉴权模式

```powershell
cd qclaw-skill
.\start_local.ps1 -Mode auth -ApiKey "your-api-key"
```

开启鉴权后，请求必须带：

```text
x-api-key: your-api-key
```

## 常见问题

### Q1：一键脚本执行后安装失败？

通常是 npm 或网络问题。先确认能执行：

```bash
npx skills --help
```

### Q2：导入成功但记忆不生效？

通常是后端未启动。请先访问 health 接口确认本地服务在线。

### Q3：如何停止服务？

在启动服务的终端按 `Ctrl + C`。
# QClaw Skill 使用说明（用户版）

本说明面向最终用户：通过仓库导入后，如何在本地快速启用长期记忆能力。

仓库地址：

- [https://github.com/youzi0812/qclaw-skill](https://github.com/youzi0812/qclaw-skill)

## 1. 导入技能

在平台中选择“从 GitHub 导入”，填入仓库地址即可。

如果平台只识别根目录技能入口，本仓库已提供根目录 `SKILL.md`。

## 2. 下载并启动本地后端

因为记忆能力由本地 API 服务提供，用户需要把仓库下载到本机并启动服务。

### Windows 最简方式（双击）

进入 `qclaw-skill` 目录，双击：

- `start_local.bat`

### PowerShell 方式

```powershell
cd qclaw-skill
.\start_local.ps1 -Mode noauth
```

说明：

- 默认端口为 `8000`
- 若 `8000` 被占用，脚本会自动尝试 `8001~8010`

## 3. 验证服务是否可用

浏览器访问：

- `http://127.0.0.1:8000/v1/health`

如果自动切到其他端口（如 `8001`），改用对应端口访问。

返回 JSON 且 `ok=true` 即代表服务正常。

## 4. 数据存储位置

本地记忆数据库：

- `qclaw-skill/data/memory.db`

只要该文件保留，同一台机器上的记忆就会持续存在。

## 5. 可选：开启鉴权模式

```powershell
cd qclaw-skill
.\start_local.ps1 -Mode auth -ApiKey "your-api-key"
```

开启后，调用方必须带请求头：

```text
x-api-key: your-api-key
```

## 6. 常见问题

### Q1：导入成功但不能用？

通常是本地后端没有启动，或端口不一致。  
先看启动终端里最终显示的 URL，再按该地址调用。

### Q2：提示端口占用？

脚本会自动换端口。  
如果 8000~8010 都占用，请手动指定更高端口：

```powershell
.\start_local.ps1 -Mode noauth -Port 9000
```

### Q3：如何停止服务？

在启动服务的终端按：

- `Ctrl + C`
